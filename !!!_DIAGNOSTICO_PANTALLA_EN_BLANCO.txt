🔍 DIAGNÓSTICO: PANTALLA EN BLANCO DESPUÉS DE SELECCIONAR USUARIO
================================================================

❌ PROBLEMA IDENTIFICADO Y CORREGIDO:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

El problema era que el hook useAuth estaba limpiando automáticamente 
el usuario guardado cada vez que se inicializaba la aplicación, lo que 
causaba que después de seleccionar un usuario, se guardara en Firebase 
pero inmediatamente se limpiara en el siguiente render.

✅ CORRECCIÓN APLICADA:
- Eliminé la limpieza automática del usuario guardado
- Ahora solo inicializa sin usuario al arrancar
- Una vez seleccionado, el usuario se mantiene durante la sesión

🔧 SI AÚN VES PANTALLA EN BLANCO:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

PASO 1: Abre DevTools (F12)
- Ve a la pestaña "Console"
- Busca errores en rojo
- Copia cualquier mensaje de error

PASO 2: Verifica la consola de PowerShell
- Mira la terminal donde corre "npm run dev"
- Busca mensajes de error

PASO 3: Recarga forzada
- Presiona: Ctrl + Shift + R
- O abre ventana de incógnito: Ctrl + Shift + N

PASO 4: Verifica la red
- En DevTools, ve a "Network"
- Recarga la página
- Verifica que no hay errores 404 o 500


🐛 ERRORES COMUNES Y SOLUCIONES:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ERROR: "Cannot read property 'uid' of null"
SOLUCIÓN: El usuario no se está guardando correctamente

ERROR: "Firebase: Permission denied"
SOLUCIÓN: Las reglas de Firestore no están bien configuradas

ERROR: "Module not found"
SOLUCIÓN: Hay un problema con los imports

ERROR: "Cannot read property 'map' of undefined"
SOLUCIÓN: Los datos no se están cargando correctamente
